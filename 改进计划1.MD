# SubTally IAP 用户体验优化计划

## 文档信息
- **版本**: 1.0
- **日期**: 2026-04-21
- **目标**: 提升 IAP 购买体验，确保 Apple 审核通过率
- **优先级**: 高

---

## 一、需求分析

### 1.1 当前问题（基于 iap-compliance-report.md）

| 问题 | 严重程度 | 影响 |
|------|----------|------|
| 购买成功无反馈 | 中 | 用户不确定购买是否成功 |
| 恢复购买无反馈 | 中 | 用户不知道恢复是否成功 |
| 错误消息未显示 | 低 | 用户无法得知错误原因 |
| 产品加载无占位符 | 低 | 价格显示延迟体验差 |

### 1.2 优化目标

1. **即时反馈**: 购买/恢复操作后立即给予用户明确反馈
2. **错误透明**: 错误信息清晰展示，帮助用户理解问题
3. **加载状态**: 产品加载期间显示合适的占位符
4. **体验一致**: 与现有 SubTally 设计风格保持一致

## 二、生成规则

### 2.1 代码编写规范

#### 规则 1: 单一职责原则
- 每个模块只负责一个功能
- View 负责展示，Manager 负责业务逻辑
- 反馈功能独立封装，不与购买逻辑耦合

#### 规则 2: 代码复用原则（三次法则）
- 优先复用现有代码
- 相同逻辑出现 3 次才抽象
- 复用现有 `PurchaseManager` 和 `PaywallView` 模式

#### 规则 3: 继承优先原则
- 使用现有 `@ObservedObject` 模式
- 继承现有 `purchaseManager` 状态管理
- 复用现有 `AppConstants` 配置

#### 规则 4: 清理废弃代码
- 修改时标记旧代码为 deprecated
- 验证新代码无影响后删除旧代码
- Git commit 注明清理范围

#### 规则 5: 风格一致性
- 使用现有 SwiftUI 风格
- 保持与 SettingsView 相同的卡片式布局
- 使用系统默认颜色和字体

### 模块 2: PaywallView 反馈增强

**需求**: 添加购买成功/失败/恢复的 Alert 提示

**文件**: `Views/Settings/PaywallView.swift`

**修改内容**:

```swift
struct PaywallView: View {
    // MARK: - 新增：反馈状态
    @State private var showSuccessAlert = false
    @State private var showRestoreAlert = false
    @State private var showErrorAlert = false
    @State private var alertMessage = ""
    
    // MARK: - 修改：购买按钮处理
    private var purchaseButton: some View {
        Button {
            Task {
                let result = await purchaseManager.purchase()
                await handlePurchaseResult(result)
            }
        } label: { /* ... */ }
        .alert("Purchase Successful", isPresented: $showSuccessAlert) {
            Button("OK") { dismiss() }
        } message: {
            Text("Thank you for upgrading to SubTally Pro!")
        }
    }
    
    // MARK: - 新增：结果处理方法
    @MainActor
    private func handlePurchaseResult(_ result: PurchaseResult) async {
        switch result {
        case .success:
            showSuccessAlert = true
        case .failed(let error):
            alertMessage = error.localizedDescription
            showErrorAlert = true
        case .cancelled, .pending:
            break
        }
    }
}
```

**检验标准**:
- [ ] 购买成功显示成功 Alert
- [ ] 购买失败显示错误 Alert（含具体错误信息）
- [ ] 恢复购买显示结果 Alert（成功/未找到购买）
- [ ] Alert 样式与系统一致

## 九、风险与回滚

### 风险
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Alert 与系统冲突 | 低 | 中 | 使用原生 .alert 修饰符 |
| 状态管理混乱 | 低 | 高 | 严格遵循 @State 使用规范 |
| 编译错误 | 低 | 高 | 小步提交，频繁构建测试 |

### 回滚计划
1. 保留原始代码分支
2. 使用 Git revert 可快速回滚
3. 关键修改分多个 commit
